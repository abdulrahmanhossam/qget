# Contributing to qget

We welcome contributions from the community! Whether you want to report a bug, suggest a feature, or submit a pull request, we're glad to have your help.

## How to Contribute

1. **Fork** the repository
2. **Create** a feature branch (`git checkout -b feature/my-new-feature`)
3. **Commit** your changes (`git commit -m "Add new feature"`)
4. **Push** to the branch (`git push origin feature/my-new-feature`)
5. **Submit** a Pull Request

## Coding Conventions

We follow standard Go coding conventions:
- Use `gofmt` for code formatting
- Add GoDoc comments for all exported functions and types
- Keep code clean, modular, and well-organized
- Write meaningful commit messages

## Getting Started

1. Clone your fork
2. Create a new branch for your feature or bugfix
3. Make your changes
4. Run `go build` to ensure the code compiles
5. Submit a PR

Thank you for contributing!
